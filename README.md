# KQL Assistant

A local, free, natural-language-to-query tool for two SIEM query languages ‚Äî
Elasticsearch (Kibana KQL) and Microsoft Sentinel (Kusto KQL) ‚Äî powered by a
locally-run LLM via [Ollama](https://ollama.com) instead of a paid API.
Nothing leaves your machine.

Describe what you're hunting for in plain English, get back a working query.
Or skip the LLM entirely and load a pre-built, deterministic "Full Hunt
Package": a set of detections mapped to MITRE ATT&CK, derived from
cybersecurity training exercise material, ready to paste into a real SIEM.

## Why this exists

Built to solve a simple problem: SQL/KQL-style query syntax is a recurring
bottleneck when you're context-switching between SIEMs during an
investigation, and every AI-assisted-query tool I could find assumed you'd
pay for a hosted LLM API. This runs entirely local ‚Äî point it at any Ollama
model (Llama 3, Qwen2.5-Coder, Mistral, etc.) and it works offline, for free,
with zero telemetry.

## Features

- **Two flavors, one tool** ‚Äî tabs for Elastic (Kibana Query Language) and
  Microsoft Sentinel (real Kusto pipe syntax), each with its own system
  prompt, field-name guidance, and few-shot examples tuned to that engine's
  actual syntax.
- **Natural language ‚Üí query** ‚Äî type a hunting request, get back a
  ready-to-paste query using standard Elastic Common Schema fields or real
  Sentinel/Log Analytics table schemas (SigninLogs, AWSCloudTrail,
  DeviceProcessEvents, etc.), generated by your local model.
- **Deterministic Full Hunt Package** ‚Äî no LLM required. Each flavor ships a
  hand-built detection set mapped to MITRE ATT&CK technique IDs:
  - **Elastic (15 detections):** C2 beaconing (including a masqueraded-jQuery
    URI trick and a Cobalt Strike-style SMB named pipe), WMIC lateral
    movement, WMI Event Subscription persistence, encoded PowerShell,
    credential-file theft, discovery, an OT/HMI port scan, an SSH pivot into
    an OT segment, and a beacon-jitter anomaly signal.
  - **Sentinel (11 detections):** contractor account compromise, Azure/Graph
    enumeration, Key Vault credential discovery, a cross-cloud AWS
    `AssumeRole` pivot, S3 document discovery (decoys vs. the real target
    file), and internal RDP lateral movement ‚Äî plus one composite query that
    reconstructs the entire attack path end-to-end in a single run.
- **100% local** ‚Äî no API keys, no per-query cost, no data sent anywhere but
  your own Ollama instance.

## Setup

### 1. Install Ollama

Download from [ollama.com](https://ollama.com), then pull a model:

```bash
ollama pull llama3
```

Other options: `llama3.1`, `mistral`, or `qwen2.5-coder` (often better at
strict structured-syntax output like KQL).

### 2. Install dependencies

```bash
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Run

```bash
python app.py
```

Open [http://localhost:5000](http://localhost:5000).

### Configuration (optional)

| Variable       | Default                  | Purpose                          |
|----------------|---------------------------|-----------------------------------|
| `OLLAMA_HOST`  | `http://localhost:11434`  | Where Ollama's API is running     |
| `OLLAMA_MODEL` | `llama3`                  | Which pulled model to use         |

```bash
OLLAMA_MODEL=qwen2.5-coder python app.py
```

## API

- `POST /generate` ‚Äî `{"query": "...", "flavor": "elastic" | "sentinel"}` ‚Üí
  `{"query": "<generated query>", "flavor": ..., "model": ...}`
- `GET /playbook?flavor=elastic|sentinel` ‚Üí full deterministic detection set
  as JSON (no LLM call, instant, reproducible)

## ‚öÝÔ∏è About the sample data

All indicators referenced in this repository ‚Äî IP addresses, domains, email
addresses, usernames, service principal names, AWS ARNs/account IDs,
hostnames, and file paths in `iocs.py`, `sentinel_iocs.py`, and both
`*_PLAYBOOK.md` files ‚Äî are **fictional artifacts from cybersecurity training
exercises** (red-team/blue-team range simulations). None of it corresponds to
real people, real organizations, or real production infrastructure. It's
included purely to demonstrate detection-engineering methodology ‚Äî mapping
adversary TTPs to MITRE ATT&CK and translating them into working hunt
queries ‚Äî not as intelligence about any actual threat actor, company, or
individual. Any resemblance to real domains, IPs, or account names is
coincidental and a byproduct of using realistic-looking training scenario
data.
